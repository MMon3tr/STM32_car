# STM32 小车项目速读总结

这份文档记录当前可运行版本的真实状态。首次打开项目时先读这里，再看源码。

## 当前状态

- 已完成左右两轮的编码器速度闭环调试，控制周期为 TIM3 的 10ms 中断。
- 当前双轮目标为 A=`2.05`、B=`2.00` 个编码器脉冲/10ms。A 是左轮；`0.05` 是用于抵消车辆轻微左偏的速度补偿。
- 两个速度环均使用增量式 PID：`P=500`、`I=10`、`D=0`。
- PID 输出测试上限为 `30000`；底层 PWM 的全量程为 `MAX_DUTY=50000`。
- 两轮独立闭环已验证可使 `N` 稳定在约 `+002`。目前正进行双轮直行微调。
- B 侧 TB6612 曾表现为 PWMB 低电平时仍持续驱动；更换/修复硬件后，B 通道已恢复对 PWM 的正常响应。

## 启动与控制流程

`user/main.c` 是当前入口：

1. 初始化 OLED、电机 PWM、编码器、USART1。
2. 初始化 A/B 速度 PID。
3. 调用 `motor_target_set(2.05f, 2.0f)` 设置左右目标速度。
4. 启动 `tim_interrupt_ms_init(TIM_3, 10, 0)`。
5. `TIM3_IRQHandler()` 每 10ms 调用 `pid_control()`。

控制链路如下：

```text
目标速度 (脉冲/10ms)
  -> motor_target_set()
  -> 增量式 PID
  -> PWM 占空比
  -> TB6612 + 电机
  -> 编码器 EXTI 脉冲计数
  -> motorA.now / motorB.now
```

`motor_target_set()` 现在接收 `float` 参数，可进行小幅左右速度补偿。正负目标值决定方向，PID 内部保存目标速度的绝对值。

## 引脚映射

| 功能 | 引脚/外设 | 说明 |
| --- | --- | --- |
| 电机 A PWM | PA0 / TIM2_CH1 | TB6612 PWMA |
| 电机 B PWM | PA1 / TIM2_CH2 | TB6612 PWMB |
| 电机 A 方向 | PA6、PA7 | TB6612 AIN1/AIN2 |
| 电机 B 方向 | PB0、PB1 | TB6612 BIN1/BIN2；代码已按实际正转校正 |
| 编码器 A | PA2 中断、PA3 判向 | EXTI2 |
| 编码器 B | PA4 中断、PA5 判向 | EXTI4 |
| 调试串口 | PA9/PA10 / USART1 | 115200 |
| OLED | PB8/PB9 | 独立软件 I2C |

TB6612 的 `STBY` 必须保持高电平，且 STM32 与电机电源必须共地。

## OLED 调试含义

正常闭环模式下：

```text
A T:0002 N:+002
A D:1 O:11960
B T:0002 N:+002
B C:0238 O:11900
```

- `T`：目标速度；当前 OLED 按整数显示，所以 A 的 `2.05` 会显示为 `0002`。
- `N`：最近 10ms 内的有符号编码器计数，是速度反馈。
- `D`：A 的逻辑方向位。
- `O`：PID 输出，范围 `0~30000`。
- `C`：TIM2 的 `CCR2`，即 B 通道实际 PWM 比较值；例如 `O=9500` 对应 `C` 约 `190`。

正常现象是 `N` 在目标附近小幅波动。若 `O` 长期到 `30000` 但 `N` 无法回到目标，优先检查负载、供电和机械阻力，而不是继续加大积分参数。

## B PWM 硬件诊断开关

`user/main.c` 中：

```c
#define B_PWM_PIN_TEST 0
```

保持 `0` 表示正常闭环运行。设为 `1` 时，程序会绕过 PID 和 TIM2，直接把 PA1 拉低并固定 B 方向输入；TB6612 正常时 B 电机必须停止。此模式仅用于排查 PWMB/驱动器硬件。
